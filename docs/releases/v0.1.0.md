# OpenGovSec Toolkit v0.1.0

Status: portfolio-ready MVP.

## Summary

OpenGovSec Toolkit v0.1.0 is the first stable portfolio milestone of the project. It provides a passive, reproducible cybersecurity and digital-governance toolkit for reviewing public-sector open-data metadata, API documentation, and repository supply-chain readiness signals.

The release is intentionally document-based and non-invasive. It reads local JSON inputs and public catalogue metadata, produces explainable findings, and generates human-readable and machine-readable outputs.

## Included modules

### Open Data Risk Scanner

- CKAN-like metadata normalization.
- Dataset metadata quality checks.
- Resource locator and format checks.
- Service-style and catalogue-style resource handling.
- Markdown and JSON report generation.
- Real public `dati.gov.it` demo data for `ambiente`, `sanita`, and `trasporti`.
- CSV aggregate summary across demo reports.

### API Security Documentation Checker

- OpenAPI/Swagger version checks.
- Path and server declaration checks.
- Security scheme checks.
- Security requirement checks.
- Operation text and operationId checks.
- Error response documentation checks.
- Contact and license metadata checks.
- Good/weak demo inputs and reports.

### Repository Readiness / Supply-Chain Readiness Checker

- License and README checks.
- Dependency manifest and lockfile checks.
- CI workflow checks.
- Community documentation checks.
- Security policy checks.
- Dependency update configuration checks.
- Release/tag metadata checks.
- SBOM declaration checks.
- Repository activity checks.
- Good/weak demo inputs and reports.

## Scope limits

This release does not perform live service testing, exploitation, vulnerability scanning, dependency resolution, or source-code security analysis.

It supports passive triage, governance review, documentation review, and portfolio demonstration. It does not certify security, compliance, PDND readiness, or software supply-chain integrity.
